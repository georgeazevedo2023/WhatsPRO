## 2. M19-S10 v2 Service Categories (tarde) — SHIPPED

### O que faz (didático)

Antes: a IA tinha pergunta "fosco ou brilho?" trancada no código — só servia pra loja de tinta. Agora: cada agente da plataforma tem **funil de qualificação editável visualmente** com etapas (stages) e pontuação (score). Multi-tenant real desbloqueado — clínica, e-commerce, RH, imobiliária, política, qualquer nicho monta seu próprio funil sem programador.

### Arquitetura final

**Hierarquia:** Categoria → Stage → Field
- **Categoria** matcha por regex em `interesse:X` (ex: `tinta|esmalte`)
- **Stage** tem `min_score`/`max_score`/`exit_action` (`search_products`/`enrichment`/`handoff`/`continue`)
- **Field** tem `score_value` que soma quando lead responde

**Como roda em runtime:**
1. Lead manda mensagem → match em categoria
2. Score atual lido da tag `lead_score:N` (default 0)
3. `getCurrentStage(score, category)` decide etapa atual
4. IA pergunta usando `phrasing` template do stage
5. Lead responde → handler `set_tags` calcula `scoreDelta`, atualiza `lead_score:N`, persiste em `lead_score_history` via RPC
6. Score atinge `max_score` do stage → `exit_action` dispara

### Onde o admin acessa

`/dashboard/ai-agent` → seleciona agente → tab **"Qualificacao"** (4ª tab, ícone ListTree)

UI tem: Cards expandíveis por categoria, drag-drop em fields, slider/inputs de score, FunnelPreviewBar colorida por exit_action, validação de regex client-side, "Restaurar Padrão", default não-removível.

## 3. PENDENTE — Próxima sessão deve focar nos testes

### A) Smoke E2E manual (PRIORITY 1)

**Objetivo:** validar que o fluxo de stages + score funciona ponta-a-ponta com agente real.

**Cenário:**
1. Abrir Helpdesk em produção (`crm.wsmart.com.br`)
2. Mandar mensagem "quero tinta para parede externa" para o número WhatsApp do agente
3. **Verificar:**
   - Agente pergunta sobre ambiente ou cor (Stage Identificação, max 30 pts)
   - Após responder "externo", checar tag `lead_score:15` e `ambiente:externo` na conversa
   - Após responder "branco", checar tag `lead_score:30` (atingiu max) e `cor:branco`
   - Sistema dispara `search_products` automaticamente (busca tinta branca externa)
   - Se busca falhar → entra em Stage Detalhamento; agente pergunta acabamento (formato: "fosco, acetinado, brilho...")
   - Tabela `lead_score_history` recebe rows com metadata (category_id, stage_id, field_key)

**Como verificar logs:**
- Edge function logs: dashboard Supabase → ai-agent
- SQL: `SELECT * FROM lead_score_history WHERE lead_id=X ORDER BY created_at DESC`
- SQL: `SELECT tags FROM conversations WHERE id=X`

### B) Agent QA Framework batch (PRIORITY 2)

Rodar batch automatizado em `/ai-agent/playground` ou via edge function `e2e-test`. **Threshold ≥80%** antes do deploy de produção mais ampla.

Foco em cenários de:
- Lead "tinta" → fluxo completo de stages
- Lead "consulta médica" → cai no default (não há categoria configurada ainda)
- Lead que responde só com tag não relacionada → score não acumula

  <action>
    For each of the 7 functions, apply these 3 mechanical transformations:

    **1. Client migration (per D-01):**
    - Remove: `import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'`
    - Remove: `const SUPABASE_URL = Deno.env.get('SUPABASE_URL')!` and `const SERVICE_ROLE_KEY = Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!` (only if used exclusively for client creation)
    - If module-level client: replace `const supabase = createClient(SUPABASE_URL, SERVICE_ROLE_KEY)` with `import { createServiceClient } from '../_shared/supabaseClient.ts'` + `const supabase = createServiceClient()`
    - If request-level client: use `createUserClient(req)` inside handler
    - IMPORTANT: If function also reads SUPABASE_URL for other purposes (e.g., passing to another fetch call), keep that env read. Only remove the ones used exclusively for createClient().

    **2. Response migration (per D-02):**
    - Add: `import { successResponse, errorResponse } from '../_shared/response.ts'`
    - Replace each `new Response(JSON.stringify({ ... }), { status: 200, headers: { ...corsHeaders, 'Content-Type': 'application/json' } })` with `successResponse(corsHeaders, { ... })`
    - Replace each `new Response(JSON.stringify({ error: '...' }), { status: 4xx/5xx, ... })` with `errorResponse(corsHeaders, '...', status)`
    - Keep `new Response(null, { headers: corsHeaders })` for OPTIONS preflight — do NOT convert
    - Keep any non-JSON responses as-is (redirects, streaming, etc.)
    - IMPORTANT: `successResponse` wraps data as `{ ok: true, ...data }`. If function currently returns `{ data: payload }`, pass `{ data: payload }` to successResponse — it becomes `{ ok: true, data: payload }`.
    - For 401 responses: use `unauthorizedResponse(corsHeaders)` from auth.ts (import if not already imported)

    **3. Logger migration (per D-02):**
    - If function does NOT already import createLogger: add `import { createLogger } from '../_shared/logger.ts'`
    - Create logger: `const log = createLogger('function-name')` (module-level for simple functions, or inside handler with requestId if available)
    - Replace `console.log('[fn] message', data)` with `log.info('message', { data })`
    - Replace `console.error('[fn] message', err)` with `log.error('message', { error: (err as Error).message })`
    - Replace `console.warn('[fn] message')` with `log.warn('message')`
    - Note: ai-agent-debounce and process-jobs ALREADY import createLogger — just migrate their remaining console.* calls

    **Special cases:**
    - `scrape-product`: may not use Supabase client at all (verify by reading file). If no client, skip step 1.
    - `group-reasons`: verify client usage before migrating.
    - `health-check`: may return custom 200/503 structures — preserve exact response shapes.
    - `e2e-test`: may have complex response structures for test results — preserve shapes.
  </action>
